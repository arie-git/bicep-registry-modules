Onboarding process
==================
| This page describes the steps of the onboarding process of new applications to the DRCP Azure platform.
| If you have any questions about this, or have an application you want to onboard to Azure, please :doc:`contact <../Need-help>` your business unit Cloud Competence Center.

Kick-off meeting
----------------
The start of the onboarding to Azure is a kick-off meeting. The goal of this meeting is to align the expectations of the DevOps team and the CCC about the DRCP platform and the onboarding process. Therefor the CCC will explain these subjects in a session and for the DevOps team this is the time to ask question about the further process.

Application assessment
----------------------
An application assessment is a comprehensive evaluation of an application's technical and functional characteristics. The assessment aims to understand the application's current state, identify improvement areas, and determine the best approach for cloud migration or modernization. It typically covers architecture, security, performance, scalability, compatibility, and compliance needs. The assessment results guide decision-making for the application's future and help organizations plan successful cloud migration or modernization projects.

Skill assessment
----------------
The use of the Azure platform is setup in such a way that it remains flexible for the DevOps teams. This comes with certain risks for APG. Part of mitigating these risks is to make sure the DevOps teams using the platform are mature enough to be responsible. To determine the maturity of the team a skill assessment is part of the onboarding. The assessment evaluates different aspects of the team including Azure skills, DevOps practices, architecture, and APG skills. For every category the team, together with the CCC, determine the maturity they have and the CCC advises the team on possible learning paths. It's the responsibility of the team to learn the necessary skills during the onboarding to deploy and manage their application in production.

Quality gates
-------------
For onboarding an application to Azure a couple of quality gates are in place for the DevOps team before they can continue to the next steps. These quality gates relate to the usages of the DTAP street. The following table shows the needs for using each of the usages.

+---------------------------------------+------------------------------------------------------------------------------------+
| Usage                                 | Needs                                                                              |
+=======================================+====================================================================================+
| Development                           | - DevOps team completed the application assessment in collaboration with the CCC   |
|                                       | - Skill assessment allows for development usage                                    |
|                                       | - Kick-off meeting completed                                                       |
+---------------------------------------+------------------------------------------------------------------------------------+
| Test                                  | - Skill assessment allows for test usage                                           |
|                                       | - The use of alpha or beta component isn't allowed                                 |
+---------------------------------------+------------------------------------------------------------------------------------+
| Acceptance                            | - Skill assessment allows for acceptance usage                                     |
+---------------------------------------+------------------------------------------------------------------------------------+
| Production                            | - Skill assessment allows for production usage                                     |
|                                       | - DHT approved the application design                                              |
+---------------------------------------+------------------------------------------------------------------------------------+
